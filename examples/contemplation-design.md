# 沉思工具设计案例

## 项目背景

创建一个沉浸式语录浏览工具，支持多主题切换和 WebGL 特效背景，让用户在欣赏视觉艺术的同时获得心灵启迪。

## 设计过程

### 1. 需求探索（使用 brainstorming skill）

**初始需求**：
- 用户想要一个可以放松心情的工具
- 需要视觉特效和文字内容结合
- 支持多主题切换

**头脑风暴结果**：
- 确定"沉思"作为核心概念
- 选择语录作为内容载体
- 使用 WebGL 特效增强沉浸感
- 初期规划 4 个主题，后扩展至 10 个

### 2. 技术选型

**WebGL 库选择**：
- ❌ Three.js：功能强大但体积过大（~600KB）
- ✅ OGL：轻量级 WebGL 库（~50KB），适合简单的全屏特效

**状态管理选择**：
- ❌ Redux：功能完整但过于重量级
- ✅ Zustand：轻量级状态管理（~1KB），API 简洁

**样式方案选择**：
- ✅ Tailwind CSS：原子化 CSS，按需生成，体积小

### 3. 主题设计

**初始 4 个主题**：
1. 🌌 宇宙（Galaxy）- 黑洞穿梭效果
2. ⚡ 电磁（Ferrofluid）- 铁磁流体效果
3. 💻 数字世界（LetterGlitch）- 黑客帝国效果
4. 🔄 轮回（Balatro）- 流体光影效果

**扩展至 10 个主题**：
5. 💡 光线（ColorBends）- 色彩弯曲效果
6. 📐 规则（Beams）- 光束效果
7. ⏳ 时间（Hyperspeed）- 超光速效果
8. ✨ 灵感（Iridescence）- 虹彩效果
9. 🌿 自然（Aurora）- 极光效果
10. 👁️ 混沌（EvilEye）- 邪眼效果

**语录设计**：
- 每个主题 10 条语录
- 中英双语支持
- 混合风格：经典名言 + 原创诗意
- 语录顺序精心设计（第一条最具代表性）

### 4. 交互设计

**语录卡片交互**：
- 点击卡片任意位置 → 下一条语录
- 左右箭头按钮 → 精确控制上一条/下一条
- 触摸滑动（>50px 水平）→ 切换语录
- 点击收起按钮 → 折叠卡片，进入沉浸模式

**主题切换**：
- 左侧纵向滚动条，10 个主题图标
- 点击切换，带淡入淡出过渡动画（500ms）
- 当前主题高亮显示

**折叠/展开**：
- 展开状态：显示语录卡片
- 折叠状态：显示底部圆形灵感图标（💡）
- 折叠后可滑动屏幕与背景互动

### 5. 移动端适配

**布局调整**：
- 主题切换器从右侧移到左侧（避开右侧无障碍按钮）
- 顶部控制栏从左上角移到顶部居中
- 语录卡片宽度从 340px 缩至 300px
- 字体从 text-lg 缩至 text-base

**安全区域**：
- 使用 `100dvh` 替代 `100vh`
- 使用 `env(safe-area-inset-*)` 适配刘海屏
- 底部按钮避开固定导航栏

**WebGL 适配**：
- 所有 Shader 改用 `gl_FragCoord` 归一化
- 移除 `100vmax` 正方形容器
- 添加 `uScale` 参数控制缩放
- 各主题独立调优缩放值

### 6. 性能优化

**代码分割**：
```typescript
// 动态导入工具组件
const Contemplation = lazy(() => import('../components/tools/Contemplation'));
const Divination = lazy(() => import('../components/tools/Divination'));
```

**Vite 分包**：
```typescript
manualChunks: {
  'vendor-react': ['react', 'react-dom'],
  'vendor-state': ['zustand'],
  'vendor-webgl': ['ogl'],
}
```

**优化效果**：
- 主包体积：1018KB → 519KB（减少 49%）
- 沉思工具单独打包：~91KB
- 第三方库单独打包，利用浏览器缓存

### 7. 新手引导

**引导内容**：
1. 🎨 主题切换：左侧图标点击切换不同主题背景
2. ←→ 语录切换：左右滑动或点击箭头切换语录
3. ⬇️ 折叠展开：点击下方按钮折叠语录卡片
4. 👆 互动特效：折叠后滑动屏幕，让画面动起来

**引导逻辑**：
- 首次访问时显示
- 点击"我知道了"或背景关闭
- 使用 localStorage 记录，只显示一次

## 关键决策

### 决策 1：选择 OGL 而非 Three.js

**原因**：
- 项目只需要简单的全屏 Shader 效果
- Three.js 功能过于强大，体积过大
- OGL 更轻量，学习成本低

**结果**：
- WebGL 库体积从 ~600KB 降至 ~50KB
- 首屏加载时间显著减少

### 决策 2：主题切换器放在左侧

**原因**：
- 右下角有固定的无障碍按钮（`fixed bottom-24 right-4`）
- 右侧会与无障碍按钮冲突
- 左侧空间更充裕

**结果**：
- 避免了 UI 元素重叠
- 用户操作更顺畅

### 决策 3：语录卡片支持三种交互方式

**原因**：
- 不同用户有不同的操作习惯
- 点击适合快速浏览
- 箭头适合精确控制
- 滑动适合沉浸式体验

**结果**：
- 用户体验更灵活
- 满足不同场景需求

## 技术难点与解决方案

### 难点 1：WebGL 效果在移动端过度放大

**问题**：
- PC 端显示正常
- 移动端显示时，效果被放大 5-10 倍

**根因**：
- 使用 `vUv` + contain 模式修正宽高比时，x/y 写反

**解决方案**：
- 改用 `gl_FragCoord` 标准归一化
- 详细记录见：[WebGL 移动端适配踩坑](../pitfalls/webgl-mobile-adaptation.md)

### 难点 2：滚动面板回弹问题

**问题**：
- 向下滚动解签面板时，面板会弹回顶部

**根因**：
- `transform: scaleY()` 动画 + `both` 填充模式重置滚动位置

**解决方案**：
- 改用纯 `opacity` 动画
- 详细记录见：[移动端布局踩坑](../pitfalls/mobile-layout.md)

### 难点 3：主包体积过大

**问题**：
- 首屏加载时间超过 5 秒
- 移动端网络下几乎无法使用

**解决方案**：
- 动态导入工具组件
- Vite 分包配置
- 详细记录见：[性能优化踩坑](../pitfalls/performance.md)

## 项目成果

**功能成果**：
- 10 个主题，100 条双语语录
- 完整的交互系统（点击、箭头、滑动、折叠）
- 新手引导，降低学习成本

**性能成果**：
- 主包体积减少 49%
- 首屏加载时间 < 3s
- 移动端体验流畅

**用户体验**：
- 沉浸式的视觉体验
- 灵活的交互方式
- 完善的移动端适配

## 经验总结

1. **技术选型要权衡**：不追求功能最全，而选择最适合的
2. **移动端优先**：所有设计决策优先考虑移动端
3. **性能至上**：每个技术选型都要考虑性能影响
4. **用户中心**：多种交互方式满足不同用户需求
5. **踩坑记录**：详细记录踩过的坑，避免重复犯错
