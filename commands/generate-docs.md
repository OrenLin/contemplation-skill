# /generate-docs 命令

## 命令描述

为项目生成完整的产品文档体系，包括 README、产品介绍、使用文档等。

## 触发条件

用户输入 `/generate-docs` 或明确要求"生成文档"、"写文档"。

## 执行流程

### 1. 分析项目信息

收集项目的关键信息：
- 项目名称和描述
- 核心功能列表
- 技术栈
- 安装和运行方式
- 使用示例
- 截图/演示链接

### 2. 确定文档类型

根据项目类型生成对应的文档：

#### 2.1 基础文档（所有项目）
- **README.md** - 项目介绍、快速开始、功能特性
- **CHANGELOG.md** - 版本历史和变更记录
- **LICENSE** - 开源协议（如需要）

#### 2.2 Web 应用文档
- **使用指南** - 用户操作说明
- **部署文档** - 部署步骤和配置
- **API 文档** - 接口说明（如有后端）

#### 2.3 库/SDK 文档
- **安装指南** - 安装和配置
- **API 参考** - 完整的 API 文档
- **示例代码** - 使用示例和最佳实践

#### 2.4 产品文档
- **产品介绍** - 面向用户的产品说明
- **功能说明** - 详细的功能介绍
- **FAQ** - 常见问题解答

### 3. 生成 README.md

使用 [templates/readme.md](../templates/readme.md) 模板：

```markdown
# 项目名称

简短的项目描述（1-2 句话）

## 特性

- 特性 1
- 特性 2
- 特性 3

## 快速开始

### 安装

```bash
npm install package-name
```

### 使用

```javascript
// 代码示例
```

## 文档

详细文档请访问 [文档链接](./docs/)

## 贡献

欢迎提交 Issue 和 Pull Request。

## 许可证

MIT
```

### 4. 生成产品介绍文档

使用 [templates/product-intro.md](../templates/product-intro.md) 模板：

```markdown
# 产品名称

## 产品概述

产品定位和核心价值主张

## 目标用户

- 用户群体 1
- 用户群体 2

## 核心功能

### 功能 1
功能描述和使用场景

### 功能 2
功能描述和使用场景

## 使用场景

场景 1：描述具体使用场景
场景 2：描述具体使用场景

## 产品优势

- 优势 1
- 优势 2
- 优势 3
```

### 5. 生成使用文档

使用 [templates/usage-guide.md](../templates/usage-guide.md) 模板：

```markdown
# 使用指南

## 快速开始

### 第一步
操作步骤

### 第二步
操作步骤

## 功能详解

### 功能 A
详细说明和使用方法

### 功能 B
详细说明和使用方法

## 常见问题

### 问题 1
解决方案

### 问题 2
解决方案
```

### 6. 生成 CHANGELOG.md

使用 [templates/changelog.md](../templates/changelog.md) 模板：

```markdown
# 更新日志

## [版本号] - 日期

### 新增
- 新功能 1
- 新功能 2

### 优化
- 优化项 1
- 优化项 2

### 修复
- 修复问题 1
- 修复问题 2
```

### 7. 生成 API 文档（如适用）

使用 [templates/api-docs.md](../templates/api-docs.md) 模板：

```markdown
# API 文档

## 概述

API 简介和使用说明

## 认证

认证方式说明

## 接口列表

### 接口 1

**请求**
```
POST /api/endpoint
```

**参数**
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| param1 | string | 是 | 参数说明 |

**响应**
```json
{
  "code": 200,
  "data": {}
}
```
```

### 8. 组织文档结构

创建文档目录结构：

```
project/
├── README.md
├── CHANGELOG.md
├── LICENSE
└── docs/
    ├── product-intro.md
    ├── usage-guide.md
    ├── api-docs.md
    └── faq.md
```

### 9. 验证文档完整性

检查清单：
- [ ] README.md 包含快速开始
- [ ] 所有代码示例可运行
- [ ] 链接有效
- [ ] 截图/图片存在
- [ ] 文档语言一致
- [ ] 无拼写错误

### 10. 生成文档报告

输出文档生成报告：

```markdown
## 文档生成报告

### 已生成文档
- README.md
- CHANGELOG.md
- docs/product-intro.md
- docs/usage-guide.md
- docs/api-docs.md

### 文档统计
- 总文档数：5
- 总字数：约 5000 字
- 代码示例：10 个

### 建议的后续优化
- 添加视频教程
- 补充更多使用示例
- 添加国际化支持
```

## 注意事项

1. **语言一致** - 所有文档使用同一种语言
2. **代码可运行** - 所有代码示例必须经过验证
3. **链接有效** - 检查所有内部和外部链接
4. **图片优化** - 使用合适的图片格式和大小
5. **版本同步** - 文档版本与代码版本保持一致
